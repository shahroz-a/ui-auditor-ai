# Styleguide

## Product Voice

Use plain language. Findings should describe the risk and the next action without sounding alarmist.

## UI Rules

- Prefer compact, scannable panels over marketing-style sections.
- Keep cards for actual tools, repeated items, and modal surfaces.
- Use icons for commands when the icon is familiar.
- Keep headings sized to their container.
- Ensure controls have visible focus states and disabled states.
- Support 320px width without horizontal scrolling.

## Code Rules

- Use strict TypeScript.
- Avoid `any`.
- Keep rules deterministic and easy to test.
- Avoid console logging in production code.
- Keep comments rare and useful.

## Color And Motion

The base palette uses ink, mint, coral, amber, and neutral surfaces. Motion should be subtle and must respect `prefers-reduced-motion`.
