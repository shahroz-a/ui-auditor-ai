# Security Policy

## Supported Versions

Security fixes are made against the `main` branch until the project starts publishing versioned releases.

## Reporting A Vulnerability

Please do not open a public issue for a vulnerability. Email Shahroz at shahrozmails@gmail.com with:

- A clear description of the issue.
- Steps to reproduce.
- Impact and affected versions, if known.
- Any suggested mitigation.

You can expect an acknowledgement within 72 hours.

## Security Model

The current app reads screenshots locally in the browser. It does not upload image files to a server. Future features that introduce server-side processing must document retention, redaction, and deletion behavior before release.
